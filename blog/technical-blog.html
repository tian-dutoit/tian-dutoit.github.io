Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

You can think of HTML as the skeleton of a body, it provides the basic structure for the website and has all of the content. CSS would then be the skin, hair, eye colour etc that sits over the top of the skeleton. It provides instructions on what different parts (elements) of the website looks like. The DOM is the way your web browser interprets the information provided by the HTML and CSS.

What is meant by boxifying design?

All of the different elements in a website are rectangular in shape (boxes), allowing you to easily alter the design and layout of a website by styling the different boxes or moving them around.

What is the box model?

The box model describes the box that surrounds every HTML element. The acutal content of the element is in the middle and then is wrapped in several layers that define the space around the content. Directly outside of the content is padding, then border and lastly margin.

What is grid based design?

The layout of a webpage is divided into a series of columns and rows (a grid) and this allows you to size and arrange elements in an appealing and easy way.

Whats all the hype about responsive webpages?

There are a range of devices that will be used to view your webpage and these will have different screen sizes and use different web browsers. You need to design your webpage so that it will still be be readable and functional no matter how it is viewed and this involves the layout changing in response to the users device.

What is semantic structure?

Semantic structure is writing your HTML in a way that makes the content easier understand. For instance, using tags that provide meaning and context to the reader instead of more generic tags that will achieve the same outcome (e.g. header instead of div or p). Some initial reading indicates that it is important for SEO as well as ranking places different weights on content within different tags.
