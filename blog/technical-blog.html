Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

You can think of HTML as the skeleton of a body, it provides the basic structure for the website and has all of the content. CSS would then be the skin, hair, eye colour etc that sits over the top of the skeleton. It provides instructions on what different parts (elements) of the website looks like. The DOM is the way your web browser interprets the information provided by the HTML and CSS.

What is meant by boxifying design?

All of the different elements in a website are rectangular in shape (boxes), allowing you to easily alter the design and layout of a website by styling the different boxes or moving them around.

What is the box model?

The box model describes the box that surrounds every HTML element. The acutal content of the element is in the middle and then is wrapped in several layers that define the space around the content. Directly outside of the content is padding, then border and lastly margin.

Udacity Reflection Part 2

What is grid based design?

The layout of a webpage is divided into a series of columns and rows (a grid) and this allows you to size and arrange elements in an appealing and easy way.

Whats all the hype about responsive webpages?

There are a range of devices that will be used to view your webpage and these will have different screen sizes and use different web browsers. You need to design your webpage so that it will still be be readable and functional no matter how it is viewed and this involves the layout changing in response to the users device.

What is semantic structure?

Semantic structure is writing your HTML in a way that makes the content easier understand. For instance, using tags that provide meaning and context to the reader instead of more generic tags that will achieve the same outcome (e.g. header instead of div or p). Some initial reading indicates that it is important for SEO as well as ranking places different weights on content within different tags.

Design to Web

What a responsive site is, and why responsiveness is important?

A responsive site is a website designed to change it's layout based on the screen size being used to access it. Because of the wide range of devices being used to view websites you need to make sure that the information presented is clear and the website features are functional no matter the size of the screen being used to view it. You also need to account for the different ways people will interact with a website when using different devices, for instance mobile phones use touch screens which makes things like drop-down menus difficult to use.

What is mobile first design is, and why it's important?

When designing a responsive website it's useful to begin your planning with the mobile version as this will be the simplest version of your design due to the smaller screen size and technical limitations. It is easier to add things to your simpler mobile design than it is to try and make all of different elements in a full desktop website fit onto a mobile screen, or having to work out what elements you will need to lose or change so that it works on the smaller screen.

What frameworks are, and their pros and cons?

Frameworks are packages of prewritten code for common and useful elements or page structures, allowing you to easilly add items to your website without having to do all of the coding yourself. This way you are not constantly rewritting code every time you are working on a new project and means there is some consistency across different developers as they may otherwise solve problems using different code. This blog page used the Skeleton framework for the layout and responsive design as it already had a number of preset column widths that were based on the screensize. By using a framewok you do sacrifice some flexibility and by reusing someone else's code you aren't learning how to solve that particular problem on your own.

What a wireframe is and why we use it?

A wireframe is a basic sketch of the layout of website. It should include all of the different elements that you want to include in the website but without the content and styling. This allows you to get a feel for what your website will look like before you actually begin to create it.

JavaScript Basics

How does JavaScript compare to HTML and CSS?

HTML provides the structure and layout for the website, CSS the style and JavaScript the functionality. JavaScript allows elements on the website to change and react to inputs like mouse clicks and keystrokes from the user. Think of HTML as the things that make up your body, like your skeleton and muscles, CSS as the colour of your skin and hair and JavaScript as the instructions telling your muscles how to move.

Explain control flow and loops using an example process from everyday life, for example 'waking up' or 'brushing your teeth'.

Control flow is the order in which your code will move from one part of the code to the next and works by evaluating if a certain condition has been met and then proceeding to the appropriate outcome. For instance "Am I hungry?" Yes -> Time for lunch. "No" -> Back to work.

Loops are blocks of code that will continue to run for a set amount of iterations or for as long as certain conditions are met. Using the above example, as long I am still hungry I will keep buying pieces of sushi. Once I am full, I can go back to work.

Explain the difference between accessing data from arrays and objects.

Arrays store lists of values and you access the data by calling the specific position in the list where the desired value is stored, noting that arrays are zero-index (the first position in the list is considered to be 0 not 1). As an example, if we wanted "burgers" in the following array - foodArray["sushi","burrito","burgers"], we would use foodArray[2] as it is in the third position in the list.

Objects store data as key-value pairs and instead of using the position of the data in a list, you use the specific attribute (key) to get the desired information. You can use either dot notation or square brack notation. For example, if the "person" object had the key-value pair "age: 15" you could use either person.age or person["age"] to return the value of 15.

Explain what functions are and why they are useful.

Functions are blocks of code that peform specific tasks. They are useful because you can reuse the function to repeat the task instead of having to write out the code multiple times.
